# 01 — Project Brief

This document defines the redesign strategy, audience, positioning, and non-negotiables.

`README.md` remains the source of truth. If this file conflicts with `README.md`, follow `README.md` and update this file.

## Redesign Objective

Move the portfolio away from a generic AI-coded cybersecurity aesthetic and toward a premium, professional, case-study-focused cybersecurity analyst portfolio.

## Current Design Problem

The existing site may rely too heavily on:

- Neon teal accents
- Cyber grids
- Repetitive cards
- Metric cards
- Badge walls
- Terminal details
- Dashboard-like layouts
- Hacker-themed visual language
- Generic cybersecurity wording

These elements can be useful individually, but together they create the feeling of a template.

## Desired Outcome

The redesigned site should feel:

- Modern
- Premium
- Professional
- Calm
- Restrained
- Recruiter-friendly
- Case-study focused
- Enterprise-ready

## Target Audience

Primary:

- Security hiring managers
- Recruiters
- IT/security leaders
- CISOs or senior security leaders

Secondary:

- Technical peers
- Interviewers reviewing project depth
- Cybersecurity internship or junior analyst reviewers

## Positioning Statement

Cybersecurity analyst focused on reducing operational risk through vulnerability management, secure configuration, automation, and investigation.

## Work Areas To Highlight

- Vulnerability management
- Secure configuration
- DISA STIG remediation
- PowerShell automation
- Tenable-based assessment
- Microsoft Defender investigations
- Sentinel/KQL-style analysis
- Threat hunting
- Incident response fundamentals
- Brute-force reduction / NSG and firewall controls

## Things To Preserve

- Real metrics
- Real project names
- Resume link
- GitHub link
- Contact path
- Actual work history
- Military-to-cybersecurity background
- Current cybersecurity internship experience
- Case-study credibility

## Things To Avoid

- Exaggerated experience
- Unsupported senior-level claims
- Hacker language
- Terminal gimmicks
- Fake dashboards
- Neon/glow-heavy design
- Generic “passionate about cybersecurity” wording
- “Securing the digital frontier” type slogans

## Brand Attributes

Use these:

- Calm
- Credible
- Technical
- Grounded
- Mature
- Professional
- Evidence-based
- Practical

Avoid these:

- Cyberpunk
- Hacker
- Glitch
- Matrix
- Bootcamp-template
- AI-generated
- Overly dramatic
