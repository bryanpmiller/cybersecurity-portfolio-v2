## Phase 2 Objective

Create a premium, restrained, modern visual foundation.

The design system should move the site away from neon cyber styling and toward a professional cybersecurity portfolio.

## Design System Direction

Use:

- Light-first design
- Muted enterprise palette
- Strong text contrast
- Clean typography
- Generous whitespace
- Subtle borders
- Soft shadows
- Minimal glow
- Consistent spacing
- Accessible color contrast
- Restrained dark sections

Avoid:

- Neon teal glow
- Bright cyan borders
- Cyber grid backgrounds
- Radial glow effects
- Overly saturated gradients
- Heavy dark containers
- Terminal-inspired decoration
- Dashboard-heavy styling

## Typography Direction

The typography should be the primary visual feature.

Use:

- Large hero headline
- Clear section headings
- Readable body text
- Strong line height
- Restrained labels
- Clean case-study body text

Avoid:

- Tiny dense text
- Too many uppercase labels
- Overly futuristic type styling
- Code-like type for normal content

## Component Direction

### Buttons

Use:

- Clear primary CTA
- Subtle secondary CTA
- No neon glow
- Strong hover/focus states

### Cards

Use fewer cards.

When cards are used:

- Soft border
- Subtle shadow
- No glow
- Clear heading
- Short supporting text
- Strong spacing

### Code / Query Blocks

Use clean formatting only for real technical evidence.

Allowed:

- Real PowerShell
- Real KQL
- Real commands
- Real config snippets

Avoid:

- Fake terminal prompts
- Decorative command-line text
- “Access granted” language
