# Cybersecurity Portfolio Project Brief

## Purpose

Build a professional cybersecurity portfolio website for Bryan Miller. The website will be used as a resume-linked proof-of-concept to increase hiring chances for vulnerability management, security analyst, SecOps, SOC, and technical compliance roles.

## Primary Audience

Hiring managers, recruiters, and technical reviewers evaluating Bryan for cybersecurity roles.

## Target Role Hierarchy

1. Vulnerability Management Analyst
2. Cybersecurity Analyst / Security Analyst
3. SecOps Analyst
4. SOC / Threat Detection Analyst
5. Technical GRC / STIG Compliance Analyst

## Core Message

Bryan reduces security risk through vulnerability management, remediation automation, secure configuration, and threat-focused investigation.

## Recommended Homepage Hero

Bryan Miller
Vulnerability Management & Cybersecurity Analyst

I help reduce security risk through vulnerability management, secure configuration, remediation automation, and threat-focused investigation.

## Primary Calls to Action

- View Projects
- Download Resume
- View GitHub
- Connect on LinkedIn

## Featured Project Order

1. Vulnerability Management Program Implementation
2. DISA STIG Remediation with PowerShell
3. Password Spray Threat Hunt: RDP Compromise Investigation
4. Akira Ransomware Threat Hunt

## Site Requirements

- Multi-page site
- Dark professional theme
- Light/dark toggle if practical
- Resume shown as a web page and downloadable as PDF
- Project case study pages
- Skills grouped by role relevance
- Contact page with LinkedIn, email, and GitHub only
- No backend required for version 1
- No contact form for version 1
- No blog for version 1

The portfolio website may later be listed as a resume project if it includes:
- Next.js

- Tailwind CSS
- Git-based deployment
- Security headers
- Responsive design
- Resume download
- Project case study structure
- Clean documentation
