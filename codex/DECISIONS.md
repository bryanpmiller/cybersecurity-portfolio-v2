# DECISIONS.md — Project Decisions

## Purpose

Use this file to track important decisions and why they were made.

This file lives at:

```text
codex/DECISIONS.md
```

Use this for meaningful decisions only.

Examples:

- Choosing a framework
- Choosing a database
- Changing architecture
- Adding a major dependency
- Rejecting a design direction
- Changing deployment target
- Changing security approach
- Changing API design

For routine progress updates, use `codex/WORKLOG.md`.

---

## Decision Template

```md
## YYYY-MM-DD — Decision Title

### Status

Accepted / Rejected / Superseded / Proposed

### Context

What problem or choice led to this decision?

### Decision

What did we decide?

### Reasoning

Why did we choose this?

### Alternatives Considered

- Alternative 1
- Alternative 2

### Consequences

Positive:
- Benefit 1

Negative / Tradeoffs:
- Tradeoff 1

### Follow-Up

- [ ] Follow-up task
```
