# 04_EDITORIAL_STRUCTURE_ARCHITECT.md - Editorial Structure and Page Flow

## Purpose

Use this architect when a task touches homepage flow, section order, storytelling, content hierarchy, or the transition from portfolio page to premium proof-of-work experience.

This file helps the site feel curated instead of assembled.

---

## Core Editorial Principle

The site should not read like:

```text
hero
stats
resume
projects
skills
contact
```

It should read like:

```text
positioning
proof
operational story
case studies
capabilities
credibility
action
```

The page can still use normal portfolio sections, but the user should feel a narrative arc.

---

## Desired Homepage Logic

A strong homepage should guide the visitor through:

1. Who Bryan is
2. What kind of risk/security work he does
3. Why the work is credible
4. Which projects prove it
5. Which tools/capabilities support it
6. Where to inspect deeper evidence
7. How to contact or download the resume

---

## Editorial Pacing Rules

Use contrast between section types:

- Statement sections
- Proof sections
- Case-study sections
- Capability sections
- CTA sections
- Quiet transition sections

Avoid a long sequence of equally weighted cards.

If the page feels monotonous, improve pacing by changing section structure before adding decoration.

---

## Content Density Rules

Dense sections are allowed when they are useful.

Use density for:

- Technical evidence
- Tooling groups
- Project details
- Resume snapshots
- Evidence chains

Use more breathing room for:

- Hero
- Section transitions
- Executive-facing summaries
- First-read proof
- Final CTA

---

## Section Transition Questions

Between any two major sections, ask:

- Why does this section come next?
- Does the visitor get new information?
- Does this section repeat the previous section?
- Is the page building confidence, or just listing content?
- Would a stronger heading/lead sentence make the transition clearer?

---

## Portfolio-to-Brand Shift

Round 3 should move the site from "my portfolio" to "my professional proof system."

That means:

- Projects become case studies
- Metrics become evidence
- Skills become capabilities
- GitHub links become proof paths
- Resume becomes a credibility artifact
- Contact becomes a clear next step
