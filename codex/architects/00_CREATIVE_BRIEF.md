# 00_CREATIVE_BRIEF.md - Round 3 Creative Direction

## Purpose

This file is the creative north star for Round 3 premium polish work.

Use it when a task involves overall design direction, homepage flow, visual tone, premium polish, or any decision that could make the site feel more or less refined.

Do not use this file to justify unsupported claims, fake metrics, or unnecessary visual effects.

---

## Target Experience

The portfolio should feel like a premium cybersecurity proof-of-work experience, not a generic developer portfolio.

The desired impression:

- Calm
- Precise
- Evidence-led
- Operationally mature
- Executive-readable
- Recruiter-friendly
- Technically credible
- Custom-built
- Restrained rather than flashy

The site should feel closer to:

- A refined security consultancy
- A threat intelligence workspace
- A modern analyst operating system
- A premium technical product site
- A proof-of-work briefing for hiring decision makers

The site should not feel like:

- A hacker/cyberpunk theme
- A gaming UI
- A generic AI-generated portfolio
- A clone of a SaaS landing page
- A dashboard stuffed with every possible metric
- A student project trying too hard to look technical

---

## Core Creative Principle

Substance already exists. Round 3 should improve perception, pacing, and polish around the substance.

Do not add more decoration just because a section feels basic. First evaluate:

1. Typography hierarchy
2. Spacing rhythm
3. Section pacing
4. Visual contrast
5. Proof placement
6. Motion restraint
7. Duplicate information

---

## Executive-Friendly Direction

A CEO, hiring manager, or senior security leader should understand the value quickly:

- What risk does Bryan reduce?
- What evidence supports that?
- What systems, tools, or workflows does he understand?
- Can he communicate technical work clearly?
- Does the site feel trustworthy and mature?

The site should communicate confidence without hype.

---

## Design Restraint Rules

Prefer:

- Strong hierarchy over more cards
- Whitespace over visual clutter
- Selective metrics over repeated metrics
- Editorial flow over grid density
- Real evidence over decorative visuals
- Motion that supports attention, not motion that competes for it

Avoid:

- Neon glow abuse
- Fake terminal walls
- Matrix-style backgrounds
- Repeated dashboard surfaces
- Excessive gradients
- Generic floating blobs
- Over-animated components
- Repeating the same metric or tool list in nearby sections

---

## Round 3 Goal

Round 3 should move the site from:

```text
polished cybersecurity portfolio
```

toward:

```text
premium cybersecurity proof-of-work brand experience
```

This does not require a full rebuild. It requires targeted improvements to hierarchy, editorial flow, motion, case-study storytelling, and visual consistency.
